## How to use

When you want to cite a particular part of the incident writeup, link directly to the heading anchor on GitHub, e.g.:

- `docs/bash-tool-troubleshooting.md#issue-description`
- `docs/bash-tool-troubleshooting.md#solution`
- `docs/bash-tool-troubleshooting.md#workarounds-implemented-during-outage`

GitHub’s Markdown heading anchors are generally formed by:
- lowercasing
- replacing spaces with `-`
- removing most punctuation

(If headings change, these anchors can change too—update this index accordingly.)
